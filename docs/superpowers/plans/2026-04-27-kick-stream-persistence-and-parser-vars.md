# Kick Stream Persistence And Parser Vars Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist Kick stream state/history through official webhooks and reconciliation, extend shared stream storage for multi-platform nullable data, and add Kick parser variable parity for live fields, metadata, category time, and kicks leaderboard.

**Architecture:** Reuse `channels_streams` and `streams` repository as shared current-stream storage, make stream fields nullable where platform support differs, and feed Kick rows from official `livestream.status.updated` plus `livestream.metadata.updated`. Use scheduler only for reconcile/bootstrap, then consume persisted/current data from parser variables and official kicks leaderboard API.

**Tech Stack:** Go, fx, pgx repositories, existing Kick EventSub handlers, scheduler timers, parser variables, project CLI migrations.
